# 三维扫描机器人 运动控制协议 V6.7

## 一、数据包

| 头标识 | 计数器 | 选项 | 地址 | 指令码 | 指令数据 | CRC32-STM32校验码 |
| --- | --- | --- | --- | --- | --- | --- |
| UINT32 | UINT8 | UINT8 | UINT8 | UINT8 | 至少4字节 | UINT32 |
| 'JD3D' |   |   |   |   |   | 对“计数器”、“选项”、“地址”、“指令码”、“指令数据”字段进行校验。 |

**说明：**

- 1）本协议中的整型数据为高字节在前。
- 2）头标识：头标识用于标记数据包的起始。当通信链路不稳定时，数据包开头几个字节可能会发生改变，因而校验码忽略头标识的4个字节。但在检查接收的数据包是否正确时，应该至少检查头标识的后两个字节。
- 3）计数器：上位软件为主站，三维扫描机器人设备为从站。计数器由主站对从站进行分别递增计数，从0到255之后，下一步回到0。计数器用于标记数据包。从站回复的数据包的计数器值与收到的相同。从站不会主动发出数据包。
- 4）地址：仪表的地址为1至254。地址0为保留地址，仪表首次上电之后为地址0。地址255为广播地址。
- 5）有的指令需要较长的时间执行，在执行过程中，设备不会回复收到的指令。此时需要使用“执行结果查询”指令，来不断查询指令的执行结果，直到收到回复。此设计为适应通信链路不稳定的情况。查询指令的执行结果时，需要确认对应的计数器与指令码为发出的计数器和指令码。
- 6）设备在执行一条指令的过程中，处于忙碌状态，不会回复收到的新指令。在设备侧，如果一条指令在接收完成时，设备处于忙碌状态，则此指令将被设备丢弃，视同没有收到此指令。
- 7）如果设备收到的数据包的校验码出错，则设备视同没有收到此数据包。
- 8）同一台仪表连续发出指令的间隔时间，在通信速率>=4800bps时，至少为5ms. 在通信速率小于4800bps时，至少为20ms.
- 9）主站连续发出指令的间隔时间应该至少为50ms.
- 10）设备的串口接收中断超过1个字节的时间，则视为此数据包的接收完成。具体时间与通信的波特率有关。
- 11）串口的通信参数默认为：波特率115200，1起始位，8数据位，1停止位，无校验。
串口通信可用的波特率为：

600,1200,2400,4800,9600,14400,19200,28800,38400,57600,115200,230400,460800

- 12）数据包的最大长度为1050字节。
## 二、工作方式

仪表首次上电后，会初始化所有参数，其中，地址初始化为0，串口通信速率初始化为115200.这些参数在仪表断电之后仍然保持。

仪表重新上电或重启之后，会自动进入引导模式。需要通过指令切换至工作模式，才可以成像。

## 三、寄存器

寄存器为仪表内部的闪存存储空间，断电后数据不丢失。

每个寄存器为32位，存储的数据为UINT32格式，高字节在前。

寄存器地址从0开始，范围为[0,500]。

有些动态参数也会用寄存器进行操作，但这些动态参数不在闪存中。

通过以下的指令对寄存器进行操作：

### 1-读寄存器

**发出：**

| 指令码 | 寄存器地址 | 确认码 |
| --- | --- | --- |
| UINT8 | UINT16 | UINT32 |
| 1 |   | 100 |

**返回：**

| 指令码 | 寄存器地址 | 寄存器数据 |
| --- | --- | --- |
| UINT8 | UINT16 | UINT32 |
| 1 |   |   |

### 2-写寄存器

**发出：**

| 指令码 | 寄存器地址 | 寄存器数据 |
| --- | --- | --- |
| UINT8 | UINT16 | UINT32 |
| 2 |   |   |

**返回：**

| 指令码 | 寄存器地址 | 寄存器数据 |
| --- | --- | --- |
| UINT8 | UINT16 | UINT32 |
| 2 |   |   |

**说明：**

- 1）对于读寄存器和写寄存器指令，如果返回指令中的“寄存器地址”与目标地址相同，表示执行成功。如果返回指令中的“寄存器地址”为65535，则表示执行出错，此时相应的“寄存器数据”为错误代码。
错误代码如下：

- 0-读写：成功。
- 10-读写：指定的寄存器地址不存在。
- 20-读：指定的寄存器读取失败。
- 30-写：指定的寄存器写入失败。
- 40-写：指定的寄存器数据超出允许范围。
- 50-写：指定的寄存器不允许写入。
100及以上是具体操作的出错代码：

- 100-不允许切换到成像模式。
- 200-切换工作模式出错。
### 寄存器的列表及其工作机制

### 0-设备的工作模式控制

- **权限：** 读写。
- **值：** 10-引导模式，20-成像模式，30-待机模式，40-休眠模式。
- **初始值：** 上电或重启之后为引导模式.
**说明：**

- 1）引导模式用于升级固件。设备每次上电或重启之后，会自动进入引导模式。
- 2）成像模式用于获取成像数据。
- 3）待机模式下，设备的耗电电流可以低至0.2A（24V供电）。从待机模式可以快速切换到工作模式。待机模式可以实现较低功率的电源驱动多台设备。
- 4）休眠模式下，设备的耗电电流可以低至0.1A（24V供电）.从休眠模式切换到待机模式或工作模式时，设备会自动进行校准。休眠模式可以实现较低功率的电源驱动多台设备。
- 5）通过向此寄存器写入对应的工作模式代码来切换工作模式。设备会在回复写寄存器指令之后，再切换工作模式。
工作模式的切换有如下的限制：

- A.在引导模式10下，仅可以切换到成像模式20。
- B.在成像模式20下，可以切换到其它任一模式。
- C.在待机模式30下，可以切换到其它任一模式。
- D.在休眠模式40下，可以切换到其它任一模式，但切换到成像模式20或待机模式30过程中会自动校准。

## 五、中断式指令集

中断式指令是指设备收到指令之后，立即回复，即使设备正处于校准过程或扫描过程中。

中断式指令使用特殊的数据包头，且其“计数器”字段的数值与常规数据包的“计数器”字段的数值无关联。其“计数器”字段为单独计数，对某一特定的设备，连续递增。

中断式指令的数据包结构如下：

| 头标识 | 计数器 | 选项 | 地址 | 指令码 | 指令数据 | CRC32-STM32校验码 |
| --- | --- | --- | --- | --- | --- | --- |
| UINT32 | UINT8 | UINT8 | UINT8 | UINT8 | 至少4字节 | UINT32 |
| 'JD3I' |   | 0 |   |   |   | 对“计数器”、“选项”、“地址”、“指令码”、“指令数据”字段进行校验。 |

### 获取帧扫描状态

**发出：**

| 指令码 | 确认码 |
| --- | --- |
| UINT8 | UINT32 |
| 3 | 1000 |

**说明：**

**立即返回：**

| 指令码 | 反馈 |
| --- | --- |
| UINT8 | UINT32 |
| 3 |   |

**说明：**

- **反馈：** 共32位数据，包括前后各一个UINT16数据（高位在前）。
前UINT16数值表示当前帧扫描进行到第几条线（从1开始计数）。如果数值为0，表示当前没有进行帧扫描。

后UINT16数值表示当前帧扫描总共有几条线。如果数值为0，表示当前没有进行帧扫描。
