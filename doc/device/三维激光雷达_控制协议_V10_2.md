# 三维激光雷达 控制协议 V10.2

**对应于电路板版本HW14及激光雷达KS2100**

---

## 一、数据包

| 头标识 | 数据长度 | **计数器** | **设备地址** | **指令码** | **指令数据** | **CRC32-STM32校验码** |
|--------|----------|------------|--------------|------------|--------------|----------------------|
| UINT32 | UINT16 | **UINT16** | **UINT8** | **UINT8** | **至少4字节** | **UINT32** |
| `'LIDA'` | | | | | | **对数据包的其余部分进行校验** |

**说明：**

1）数据包为通讯链路上传输的基本单位。

本协议中的整型数据为高字节在前。

数据包的最小长度为18字节，最大长度为1400字节。

其中，指令数据的长度范围为4至1386字节。

2）头标识：用于标记数据包的起始，可以用于从连续的数据流中区分数据包。

3）数据长度：后续数据的长度（包括校验码字段的长度），可用于确定数据包的结尾。

4）计数器：主机每发出一个数据包，将计数器值加1。从机回复的数据包与主机发给它的对应数据包的"计数器"值相同。

有的指令需要较长的时间执行，在执行过程中，设备不会响应收到的一些其它指令。此时需要使用6号指令"查询执行结果"，来不断查询指令的执行结果，直到收到回复。查询执行结果时，需要确认对应的计数器与指令码为发出的计数器和指令码。也可以使用5号指令"停止当前的执行"，来中断一些指令的执行过程。

设备在执行某些指令的过程中，处于忙碌状态，不会回复收到的新指令。在设备侧，如果一条指令在接收完成时，设备处于忙碌状态，则此指令将被设备丢弃，设备视同没有收到此指令。

5）设备地址：固定为0.

6）指令码：从1开始。

7）指令数据：至少为4字节。

8）校验码：CRC32-STM32格式，对数据包的其余数据进行校验。如果接收方收到的数据包的校验码出错，则接收方会丢弃此数据包。

## 二、工作方式

1. 每台设备使用TCP协议，具有一个IP地址，是TCP的服务器端。IP协议版本为IPv4.

2. 上位软件是TCP的客户端。

3. 上位软件被称为主站，设备被称为从站。

4. 设备参数：

　　激光雷达的型号：KS2100

5. 通信参数

　　设备的IP地址：

　　设备的端口：23

　　（设备为TCP服务器）

　　最大数据速率： 100k 字节/秒

　　激光雷达的光心圆弧轨迹的半径：

　　KS2100: 约25毫米

## 三、寄存器

寄存器中的一部分为设备内部的闪存存储空间，断电后数据不丢失。

每个寄存器为32位，存储的数据为UINT32或INT32格式，高字节在前。

寄存器地址从1开始，范围为[1,500]。

有些动态参数也会用寄存器进行操作，但这些动态参数不在闪存中。

通过以下的指令对寄存器进行操作：

### 1-读寄存器

发出：

| **指令码** | **寄存器地址** | **确认码** |
|------------|----------------|------------|
| **UINT8** | **UINT16** | **UINT32** |
| **1** | | **100** |

返回：

| **指令码** | **寄存器地址** | **寄存器数据** |
|------------|----------------|----------------|
| **UINT8** | **UINT16** | **UINT32** |
| **1** | | |

### 2-写寄存器

发出：

| **指令码** | **寄存器地址** | **寄存器数据** |
|------------|----------------|----------------|
| **UINT8** | **UINT16** | **UINT32** |
| **2** | | |

返回：

| **指令码** | **寄存器地址** | **寄存器数据** |
|------------|----------------|----------------|
| **UINT8** | **UINT16** | **UINT32** |
| **2** | | |

说明：

1）对于读寄存器和写寄存器指令，如果返回指令中的"寄存器地址"与目标地址相同，表示执行成功。如果返回指令中的"寄存器地址"为65535，则表示执行出错，此时相应的"寄存器数据"为错误代码。对于写寄存器指令，如果执行成功，返回的"寄存器数据"与写入的相同。

错误代码如下：

　　10-读写：指定的寄存器地址超出范围。

　　20-读：指定的寄存器读取失败。

　　30-写：指定的寄存器写入失败。

　　40-读写：指定的寄存器数据超出范围。

　　50-写：指定的寄存器不允许写入。

说明：

### 4-校准X轴电机

发出：

| **指令码** | **电机代码** |
|------------|--------------|
| **UINT8** | **UINT32** |
| | **100** |

说明：

1）电机代码：100-X轴电机。

查询结果：

1）使用"查询执行结果"指令。

2）结果：

　　当结果A为1000时，表示校准执行中。结果B为0.

　　当结果A为2000时，表示校准失败。结果B为0.

　　当结果A为4000时，表示校准成功。结果B为0.

3）查询结果时需要确认"对应的计数器值"为发出的计数器值，且"对应的指令码"为发出的指令码。

4）结果B为校准过程的时间，单位为毫秒。

### 5-停止当前的执行

发出：

| **指令码** | **操作码** |
|------------|------------|
| **UINT8** | **UINT32** |
| | **100** |

立即返回：

| **指令码** | **对应的计数器值** | **对应的指令码** | **反馈** |
|------------|-------------------|-----------------|----------|
| **UINT8** | **UINT16** | **UINT8** | **UINT8** |
| | | | **10或20或30** |

说明：

1）对于某些指令，可以使用本指令中断其执行。

2）对应的计数器值与指令码：所要停止的执行对应的计数器值与指令码。

3）反馈：

　　10-可以停止，当前的执行会在较短的时间内停止。

　　20-无法停止，当前的执行无法停止。

　　30-已经停止，执行过程在收到本指令前已经停止。

### 6-查询执行结果

发出：

| **指令码** | **操作码** |
|------------|------------|

### 7-X轴电机移动到指定的线角度

发出：

| **指令码** | **数据** |
|------------|----------|
| **UINT8** | **UINT32** |
| | 线角度 |

说明：

1）线角度的类型为UINT32，单位为：0.001度。线角度为相对校准点的偏移角度，范围为[0,190000]，对应于0至190度。实际可用精度受到机械结构的限制。

查询结果：

1）使用"查询执行结果"指令。

2）结果：

A. 当结果A为1000时，表示移动进行中。此时，结果B为0至10000，表示移动过程的进度。

B. 当结果A为[2000,2999]时，表示移动失败。

　　此时，结果A减去2000后得到错误代码。结果B为移动过程曾经的进度。

　　错误代码的含义：

　　0-未知错误。

　　1-参数错误。

　　2-X轴电机未校准。

　　3-电机驱动电路报警。

　　4-电机位置出现过量偏移。

C. 当结果A为4000时，表示移动成功。结果B为移动过程使用时间的毫秒数。

3）查询结果时需要确认"对应的计数器值"为发出的计数器值，且"对应的指令码"为发出的指令码。

### 11-获取一个扫描场

发出：　a　b　c　d　e　f

| **指令码** | X轴开始角度 | X轴结束角度 | X轴分辨率 | Y轴开始角度 | Y轴结束角度 | Y轴分辨率 |
|------------|------------|------------|----------|------------|------------|----------|
| **UINT8** | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 |
| | | | **0** | **0** | **0** | **0** |

查询结果：

1）使用"查询执行结果"指令。

2）结果：

A. 当结果A为1000时，表示获取进行中。此时，结果B为0至10000，表示获取过程的进度。

B. 当结果A为[2000,2999]时，表示获取失败。结果B为获取过程曾经的进度。

　　此时，结果A减去2000后得到错误代码。

　　错误代码的含义：

　　0-未知错误。

　　1-参数错误。

　　2-X轴电机未校准。

　　3-电机驱动电路报警。

　　4-电机位置出现过量偏移。

　　5-激光雷达错误。

C. 当结果A为3000时，表示获取过程手动中断。结果B为获取过程曾经的进度。

D. 当结果A为4000时，表示获取成功。结果B为获取数据的字节数。

　　当结果A为4001时，表示获取完成，但数据中存在空白的扫描线数据。

中断执行：

可以使用"停止执行当前指令（5号）"指令，中断其执行过程。

指令说明：

1）X、Y轴开始角度的单位为0.001度，X、Y轴结束角度的单位为0.001度，X、Y轴角度分辨率的单位为0.000001度。

X轴开始角度、结束角度的范围为[0,190000]，对应于0至190度，从校准点开始计算，取值应该为1000的整数倍。

X轴分辨率的范围为[1,190000000]。对于KS2100型激光雷达，X轴分辨率固定为0.125度。命令中对应的字段可以置为0.

2）

A. 对于KS2100型激光雷达，Y轴开始角度固定为-5度，Y轴结束角度固定为185度，Y轴分辨率固定为0.125度。命令中对应的字段可以都置为0.

3）扫描场数据的格式：

| 头标识 | 时间戳 | X轴开始角度 | X轴结束角度 | X轴分辨率 | Y轴开始角度 | Y轴结束角度 | Y轴分辨率 | 后续的校验码字段的和 | 线0的数据（包括其头标识） | 线1的数据（包括其头标识） | … |
|--------|--------|------------|------------|----------|------------|------------|----------|---------------------|--------------------------|--------------------------|---|
| UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | | | … |
| 0xaabbccdd | 毫秒 | | | | | | | | | | … |

格式说明：

A. 扫描场数据为头部附加上各条扫描线数据的连续排列。

B. 时间戳为场数据获取完成的时刻，为设备上电或复位以来经过的毫秒数。

4）扫描线数据的格式：

| 头标识 | 后续数据的校验码 | 时间戳 | X轴的线角度 | Y轴开始角度 | Y轴结束角度 | Y轴角度分辨率 | 点数据 |
|--------|-----------------|--------|------------|------------|------------|--------------|--------|
| UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | UINT32 | … |
| 0x11223344 | | 毫秒 | | | | | … |

格式说明：

A. 校验码为CRC32-STM32，为后续所有字段数据的校验码。

B. 时间戳为线数据获取完成的时刻，为设备上电或复位以来经过的毫秒数。

C. 点数据为连续排列的测距值，类型为UINT24，如果一条扫描线的点数据都为0，表示此条扫描线获取失败（KS2100型激光雷达存在偶发的整条扫描线获取失败的情况）。

单位为设备当前的测距值单位（寄存器5，对于KS2100，此单位是1毫米）。
