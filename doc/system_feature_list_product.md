# StdioLink 产品视角功能清单

## 1. 产品定位

- 面向工业设备接入、业务流程编排和运行管控的一体化平台
- 把“设备驱动开发、流程编排、项目配置、实例调度、在线调试、可视化运维”整合到同一系统
- 适合设备联调、产线集成、仿真验证、后台服务托管和现场运维

## 2. 设备接入能力

- 统一接入多种工业设备和协议
- 支持 Modbus TCP、Modbus RTU、串口 RTU、PLC、3DVision、雷达、模拟量输出等设备类型
- 以 Driver 形式封装设备通信细节，降低上层业务对具体协议的依赖
- 支持新增自定义 Driver，扩展新的设备类型和命令面
- 支持设备能力标准化建模，便于不同设备以统一方式被调用

## 3. 设备调试与联调能力

- 支持直接对 Driver 发命令并查看返回结果
- 支持实时查看执行过程中的中间事件和最终结果
- 支持在线填写命令参数并做必填校验
- 支持复用默认参数和最近一次执行参数
- 支持在浏览器中完成设备联调，无需每次手工写命令行
- 支持在线取消命令执行和清理消息流

## 4. 业务流程编排能力

- 支持把多个 Driver 组合成完整业务流程
- 支持使用 JavaScript 编写业务编排逻辑
- 支持基于配置驱动业务流程，而不是把流程逻辑全部写死在驱动里
- 支持设备间联动，例如 PLC 控制、视觉登录、事件订阅、扫描流程协调
- 支持把通用外部程序纳入平台统一管理和调度

## 5. Service 模板能力

- 支持把通用业务逻辑封装为可复用 Service 模板
- 每个 Service 都可定义自己的配置结构和运行入口
- 支持基于模板快速创建新的业务实例
- 支持按业务场景复用同一 Service，减少重复开发
- 支持内置扫描编排、通用进程执行、Modbus 服务、PLC 仿真等模板

## 6. 项目化配置能力

- 支持将 Service 实例化为 Project
- 支持为每个 Project 配置独立参数
- 支持创建、编辑、保存、删除、重载 Project
- 支持启用或禁用单个 Project
- 支持把业务逻辑与现场参数解耦，便于复制部署和多实例运行
- 支持项目配置持久化保存，方便运维交接与现场恢复

## 7. 实例运行与调度能力

- 支持手动启动和停止业务实例
- 支持一次性触发执行
- 支持固定频率周期调度
- 支持常驻守护运行与异常自动重启
- 支持统一查看当前运行实例
- 支持终止异常实例，恢复现场控制
- 支持让不同业务按不同策略运行，覆盖测试、任务型和常驻型场景

## 8. 运行监控能力

- 支持查看系统当前服务、项目、实例、驱动的统计状态
- 支持查看实例运行状态、启动时间、工作目录和进程信息
- 支持查看运行日志和事件流
- 支持实时刷新实例与调度变化
- 支持发现实例退出、失败、重启等运行情况
- 支持为开发联调和运维巡检提供统一观察入口

## 9. Web 管理台能力

- 提供 Dashboard 总览页面
- 提供 Services、Projects、Instances、Drivers 四类核心管理页面
- 提供 DriverLab 在线调试页面
- 提供项目创建、详情查看、配置编辑、实例查看等完整操作链路
- 提供基于表单的配置编辑体验，降低命令行操作门槛
- 提供多语言界面能力
- 支持浅色/深色主题切换

## 10. 对外集成能力

- 提供 REST API，方便第三方系统接入
- 提供 SSE 实时事件流，适合做状态看板和日志订阅
- 提供 WebSocket 调试通道，适合浏览器端实时双向交互
- 支持前后端分离架构，便于扩展 WebUI 或其他客户端
- 支持通过标准 HTTP 接口融入现有平台、MES、调试工具或自动化脚本

## 11. 典型内置业务能力

- 料箱扫描编排服务：协调 PLC 与视觉设备完成扫描业务
- 通用进程执行服务：统一托管外部程序并输出实时日志
- Modbus TCP 从站服务：为外部主站提供 TCP 从站能力
- Modbus RTU over TCP 从站服务：提供 RTU 帧风格的 TCP 从站能力
- Modbus RTU 串口从站服务：提供串口从站能力
- PLC 升降装置仿真服务：用于业务联调、验证和演示

## 12. 典型用户价值

- 对开发人员：降低新增设备驱动、流程编排和联调成本
- 对实施人员：减少现场脚本散落、工具分裂和手工操作
- 对运维人员：统一查看项目、实例、日志和设备调试状态
- 对项目管理者：提升设备接入复用率和业务部署标准化程度
- 对整体产品：把设备接入工具升级为可管理、可调度、可扩展的平台能力
