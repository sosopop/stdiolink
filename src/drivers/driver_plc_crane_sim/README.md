# driver_plc_crane_sim

## 1. 协议寄存器映射（参考 `doc/device/plc_crane.md`）

### 保持寄存器（写控制）

- `HR[0]` 气缸动作：`0=停止`，`1=上升(收缩)`，`2=下降(伸出)`
- `HR[1]` 阀门动作：`0=停止`，`1=打开`，`2=关闭`
- `HR[2]` 启停：`0=停止`，`1=启动`（当前仅保存位值）
- `HR[3]` 模式：`0=手动`，`1=自动`

### 状态反馈（保持寄存器）

- `HR[9]=气缸上到位`
- `HR[10]=气缸下到位`
- `HR[13]=阀门开到位`
- `HR[14]=阀门关到位`

## 2. 默认初始状态

- 阀门默认关闭：`valve_state=closed`
- 气缸默认上到位（收缩态）：`cylinder_state=at_top`
- 对应状态位：`HR9=1`、`HR10=0`、`HR13=0`、`HR14=1`

## 3. 状态机与延时

### 气缸状态机

- `HR[0]=1`：进入 `moving_up`，经过 `cylinder_up_delay` 到 `at_top`
- `HR[0]=2`：进入 `moving_down`，经过 `cylinder_down_delay` 到 `at_bottom`
- `HR[0]=0`：若在移动中转 `idle`，并停止当前移动定时器

### 阀门状态机

- `HR[1]=1`：进入 `moving_open`，经过 `valve_open_delay` 到 `open`
- `HR[1]=2`：进入 `moving_close`，经过 `valve_close_delay` 到 `closed`
- `HR[1]=0`：若在移动中转 `idle`，并停止当前移动定时器

## 4. 自动/手动控制逻辑

### 手动模式（`HR[3]=0`）

- 可独立写 `HR[0]` 与 `HR[1]`，分别控制气缸和阀门动作

### 自动模式（`HR[3]=1`）

- 切到自动后立即触发组合动作：
1. 阀门执行打开（等价内部执行 `HR[1]=1`）
2. 阀门开到位后，气缸执行下降伸出（等价内部执行 `HR[0]=2`）
3. 气缸下到位后停在目标态

- 自动模式下拒绝外部手动写 `HR[0]`/`HR[1]`，返回错误并保持当前规范值

## 5. Driver 运行与事件

- Driver 仅提供 `run` 命令，启动后通过 ModbusTCP 通讯
- `heartbeat_ms`：
  - 默认 `0`（关闭心跳）
  - `>0` 时按周期发送 `sim_heartbeat` 事件
- `run` 成功后发送 `started` 事件，不发送 `done`（可在 oneshot 模式保持事件流）
